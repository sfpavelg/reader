# UX для детей 6–7 лет

## Минимум текста в интерфейсе

Меню — **иконки и озвучка** (голосовой помощник объяснит правила позже). Ребёнок не должен читать инструкцию, чтобы начать играть.

## Крупно и с «воздухом»

- Большие зоны нажатия (удобно для пальца, не для точного стилуса).
- Чёткие границы кнопок — важно на слабых экранах с низким контрастом.
- Один главный объект на экране в момент задания.

## Тактильный отклик

Лёгкая **вибрация** при правильном / неправильном действии (можно отключить в настройках). Звук — **короткий**, не пугающий.

## Ошибка — это подсказка, не провал

- **Нет** таймера «на поражение».
- **Нет** громкого «бузз» и красного креста на весь экран.
- Есть **мягкая подсказка**: повтор, чуть больше времени, второй шанс.

Таймер уместен только там, где он **часть упражнения** (вспышка, бегущая строка) — и всегда **регулируемый**.

## Слабые планшеты

Мы сознательно избегаем тяжёлой анимации, фоновой музыки по умолчанию и огромных картинок. Таблицы Шульте **генерируются из текста**, а не хранятся как сотни PNG.

## Выход без потери

Ребёнок может **выйти в любой момент** — микро-сессия сохранится. Это снижает тревогу «мама заставила дочитать до конца» и повышает доверие к приложению.
