# Fooocus: локальная генерация ассетов

Fooocus нужен как локальная мастерская изображений для персонажей, слоёв и эффектов Reader. Он работает на компьютере пользователя, поэтому нет лимитов на количество попыток, кроме времени и мощности видеокарты.

## Для чего используем

- Генерация слоёв персонажей: тело, глаза, рот, эмоции.
- Генерация эффектов: звезда, комета, искры.
- Подбор визуального стиля перед подключением ассетов во Flutter.
- Много быстрых попыток без кредитов и онлайн-лимитов.

Для production-ассетов всё равно нужен контроль: выбирать удачный результат, удалять фон в редакторе, сохранять файл в правильную папку проекта.

## Железо

Текущая видеокарта: **NVIDIA GTX 1060 6 GB**.

Рекомендации:

- Генерировать по 1 изображению за раз.
- Начинать с 1024×1024 или меньше.
- Не ставить большой batch.
- Если появляются ошибки памяти, уменьшить размер изображения или включить low VRAM режим, если он доступен в сборке.
- Первая генерация и первый запуск могут быть медленными из-за скачивания моделей и прогрева.

## Установка на Windows

1. Скачать Fooocus с официального репозитория:

   <https://github.com/lllyasviel/Fooocus>

2. Распаковать архив, например:

   ```text
   C:\AI\Fooocus
   ```

3. Запустить:

   ```text
   run.bat
   ```

4. При первом запуске Fooocus скачает модели. Это может занять долго и потребовать несколько гигабайт на диске.

5. После запуска откроется локальный веб-интерфейс. Обычно адрес такой:

   ```text
   http://127.0.0.1:7865
   ```

Если браузер не открылся сам, адрес можно открыть вручную.

## Батники в папке

В распакованной папке обычно есть несколько файлов запуска:

### `run.bat`

Обычный универсальный запуск Fooocus.

Использовать по умолчанию для наших задач: персонажи, слои, рот, глаза, звёзды, ассеты приложения.

### `run_anime.bat`

Запуск с аниме-настройками/пресетом. Обычно он подбирает стиль, модель или параметры под anime/illustration.

Подходит, если нужен рисованный аниме-стиль. Для Колобка пока не основной вариант, потому что у нас мягкий 3D-cartoon стиль.

### `run_realistic.bat`

Запуск с реалистичными настройками/пресетом. Обычно ориентирован на реалистичные фото/кинематографичный вид.

Для Колобка лучше использовать осторожно: может увести персонажа в реалистичную глину, кожу, лесные сцены и лишнюю детализацию. Для детского маскота это не всегда подходит.

## Как генерировать ассеты для Колобка

Проектная папка:

```text
assets/characters/kolobok/
```

Промты хранятся здесь:

```text
assets/characters/kolobok/prompts/
```

Рабочие ассеты взрослого Колобка:

```text
assets/characters/kolobok/stage_06_adult/
  body.png
  eyes_open.png
  eyes_wink.png
  mouth_smile.png
```

Эффекты:

```text
assets/characters/kolobok/effects/
  star.png
  comet.png
```

## Рабочий процесс

1. Берём промт из `assets/characters/kolobok/prompts/...`.
2. Прикладываем только нужные референсы.
3. Генерируем несколько вариантов.
4. Выбираем самый чистый слой.
5. Если фон белый, удаляем его онлайн-редактором позже.
6. Сохраняем в правильное место проекта.
7. Подключаем или тестируем в Flutter.

## Важные правила промтов

Для слоёв персонажа обязательно повторять:

```text
one asset only
mouth only / eyes only / body only
pure white background
no full character
no scene
no text
no labels
no collage
centered
easy to remove background
```

Если генератор начинает рисовать целого персонажа, нужно убрать лишние референсы. Например, для рта лучше прикладывать только картинку рта или лица с нужной улыбкой, но не `body.png`.

## Интерфейс Fooocus

### Основное поле промта

В нижнее большое текстовое поле вводится основной запрос. Для ассетов Колобка сюда вставляем текст из файлов:

```text
assets/characters/kolobok/prompts/
```

Если нужен строгий слой, промт должен явно запрещать лишнее:

```text
mouth only
no face
no body
pure white background
one asset only
no collage
```

### Кнопка «Сгенерировать»

Запускает генерацию. На GTX 1060 6 GB лучше генерировать по 1 изображению за раз и не включать тяжёлые режимы качества без необходимости.

### «Входное изображение»

Галочка включает режим работы с референсом. После включения появляется область, куда можно загрузить картинку.

Использовать для:

- повторения стиля;
- извлечения/перерисовки слоя;
- генерации вариаций по существующему ассету.

Для точных слоёв лучше прикладывать как можно меньше референсов. Например:

- для рта — только лицо с нужной улыбкой или сам рот;
- для подмигивания — только `eyes_open.png`;
- для тела — только нужный body reference.

Если приложить `body.png` при генерации рта, Fooocus может снова нарисовать целого персонажа.

### Вкладка «Улучшение или изменение тональности»

Это image-to-image режим: Fooocus берёт входную картинку и меняет её.

Настройки изменения:

- **Неполноценный** — слабое изменение, ближе всего к исходнику. Хорошо для аккуратного улучшения слоя.
- **Различаются (незначительно)** — среднее изменение. Можно пробовать для вариаций, но уже выше риск ухода от референса.
- **Различный (сильный)** — сильное изменение. Для наших слоёв часто опасно: может нарисовать целого персонажа или новый стиль.

Для ассетов Колобка начинать с:

```text
Неполноценный
```

или:

```text
Различаются (незначительно)
```

### Кнопки масштаба и качества

В этой вкладке могут быть варианты:

- **Увеличение масштаба (1.5 раза)** — апскейл изображения.
- **Увеличение масштаба (в 2 раза)** — более сильный апскейл.
- **Улучшенное качество (в 2 раза)** — улучшение качества, медленнее.
- **Улучшенное качество (быстрее в 2 раза)** — более быстрый вариант улучшения.

Для первого прогона ассета их лучше не включать. Сначала нужно получить правильную форму. Улучшать качество стоит только после выбора удачного варианта.

### Вкладка «Изображение-подсказка»

Используется как reference/image prompt: картинка влияет на стиль, форму или композицию.

Полезно, когда нужно:

- сохранить стиль Колобка;
- дать пример глаз;
- дать пример улыбки;
- удержать общую форму.

Но если референс слишком полный, модель может повторить всё изображение. Поэтому для слоя рта лучше прикладывать не тело, а рот/лицо с нужной улыбкой.

### Вкладка «Заполнение или удаление старой краски»

Это inpaint/outpaint. Полезно, когда нужно изменить только часть изображения.

Примеры:

- удалить фон;
- перерисовать только рот;
- убрать лишние глаза/тело;
- заменить одну деталь, не трогая остальное.

Для точного извлечения рта из лица это может быть полезнее обычного image-to-image, но требует аккуратной маски.

### Вкладка «Описывать»

Обычно используется для описания картинки текстом. Может помочь получить промт по уже удачному изображению.

### Вкладка «Усиливать»

Обычно связана с enhance/upscale: улучшение качества, детализация, увеличение.

Использовать после того, как найден правильный ассет.

### Вкладка «Метаданные»

Показывает параметры генерации, если они сохранены в изображении. Полезно, чтобы повторить удачный результат.

## Рекомендованные настройки для слоёв Колобка

Для рта:

- включить **Входное изображение**;
- приложить `face_smile_reference.png` или рот-референс;
- режим: **Неполноценный** или **Различаются (незначительно)**;
- не включать сильное изменение;
- не включать апскейл до выбора удачного варианта.

Для глаз:

- приложить только `eyes_open.png`;
- просить поменять только один глаз;
- не прикладывать `body.png`, чтобы модель не рисовала персонажа целиком.

Для эффектов:

- можно использовать обычный prompt без референса;
- QWEN/Fooocus хорошо подходят для `star`, `comet`, `sparkle`.

## Когда использовать другие режимы

- `run.bat` — основной режим для Reader.
- `run_anime.bat` — если специально нужен аниме/иллюстративный персонаж.
- `run_realistic.bat` — если нужны реалистичные тесты, но для Колобка чаще мешает.

Для текущего стиля Колобка начинать с `run.bat`.

## Как остановить Fooocus

Закрыть вкладку в браузере недостаточно: браузер — это только клиент, а сам Fooocus продолжает работать в окне `cmd`.

Правильная остановка:

1. Закрыть вкладку Fooocus в браузере.
2. Перейти в окно `cmd`, где запущен Fooocus.
3. Нажать:

   ```text
   Ctrl + C
   ```

4. Если появится вопрос:

   ```text
   Terminate batch job (Y/N)?
   ```

   нажать:

   ```text
   Y
   ```

5. После этого можно закрыть окно `cmd`.

Просто закрыть окно `cmd` крестиком обычно тоже остановит процесс, но аккуратнее завершать через `Ctrl + C`.

Проверка: после остановки адрес `http://127.0.0.1:7865` уже не должен открываться.
